# CLAUDE.md

## Invariants

1. **All critic agents must respond with exactly one of: (a) the word "APPROVE" with no additional text, or (b) "REJECT" followed by reasons.** There is no "approval with comments" — if a critic has feedback, it must REJECT. Violation: a critic response contains approval alongside commentary, suggestions, or caveats. (Text-only: no instrumentation exists to parse or validate critic responses programmatically. A test becomes feasible when the harness gains critic-response validation.)
2. **For every work request, the lead agent must invoke `/enhance-harness` or `/new-sdlc` before beginning execution. If the agent cannot determine which skill applies, it must ask the user to disambiguate.** Violation: the agent produces deliverables (files, commits, mission.md) for a work request without having first invoked one of the two routing skills, or proceeds despite ambiguity without asking the user. (Text-only: no instrumentation exists to log skill invocations or assert invocation order; the harness currently has no test infrastructure for agent conversation behavior. A test becomes feasible when the harness gains session-level observability.)
3. **Invariants are best captured as tests, not text.** Where possible, encode invariants as automated tests rather than prose rules. Text invariants are a last resort when a test is infeasible.
4. **Existing tests are invariants.** Modifying or deleting existing test code is an invariant change. Adding new tests is not. Violation: a commit diff modifies or deletes existing test code without following the invariant-change process (2-critic review, normal flow). (Text-only: no CI or pre-commit hook exists to detect test-file modifications automatically. A test becomes feasible when the harness gains a pre-commit check that flags diffs touching existing test files.)
5. **On session start, the lead agent must check for an existing `handoff.md` at the project root. If it exists but appears irrelevant to the dev's current prompt, the agent must confirm with the dev before proceeding.** Violation: the agent begins work on a new request without checking for `handoff.md`, or silently discards a relevant `handoff.md` without confirmation. (Text-only: no instrumentation exists to detect whether the agent checked for `handoff.md` at session start. A test becomes feasible when the harness gains session-level observability.)

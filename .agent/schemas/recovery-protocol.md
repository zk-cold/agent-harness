# Governance Scope
This document applies only when `handoff.md` records the mission in aborted state.

## Blocker Removal
Recovery must not begin unless the user provides input that addresses the blocker summary.

## Reinstatiation
Mission phase is reset to `Mission Creation` after Blocker Removal.

## Reuse Worktree
Recovery reuses the existing worktree. Any governed document by `governance-schema.md` must be reset

# Considerations

## Previous Mission
The previous `mission.md`, if present on worktree, is regarded as a draft that may demand modifications before critic review(s).
